### High-level Summary

This week the team shipped `hydra-node` `2.1.0` - a significant release that
brings improved deposit security, the new SQLite-backed event store, a ~7%
reduction in snapshot confirmation latency, and compatibility with
`cardano-node` protocol version 12+. We encourage all operators to upgrade.
Alongside the release, work continues on two long-standing items: partial
fanout and a user-friendly configuration file for `hydra-node` to replace the
current sprawl of command-line flags.

### What did the team achieve?

* Released `hydra-node` `2.1.0`, which includes improved deposit security, the SQLite-backed event store, a ~7% reduction in snapshot confirmation latency, and compatibility with `cardano-node` protocol version 12+ [release 2.1.0](https://github.com/cardano-scaling/hydra/releases/tag/2.1.0)
* Upgraded to `cardano-api` 11.1 / `cardano-node` 11.0.1 to support protocol version 12+ [#2607](https://github.com/cardano-scaling/hydra/pull/2607)
* Switched from a lazy to a strict `Map` to reduce memory pressure during transaction processing [#2599](https://github.com/cardano-scaling/hydra/pull/2599)

### What's next?

* Continue work on partial fanout — choosing the UTxO set to fan out and final code review [#2593](https://github.com/cardano-scaling/hydra/pull/2593), [#2324](https://github.com/cardano-scaling/hydra/pull/2324), [#2573](https://github.com/cardano-scaling/hydra/issues/2573)
* Reduce disk usage in `hydra-node` [#2601](https://github.com/cardano-scaling/hydra/pull/2601), [#2600](https://github.com/cardano-scaling/hydra/issues/2600)
* Investigate a node observing/persisting deposit events from unrelated Heads [#2606](https://github.com/cardano-scaling/hydra/issues/2606)
* User-friendly `hydra-node` configuration via config file [#2581](https://github.com/cardano-scaling/hydra/pull/2581)
* Accept `PaymentExtendedKey` (HD wallet keys) [#2583](https://github.com/cardano-scaling/hydra/pull/2583)
