The Hydra team kicks off the new year by with a reinvigorated focus on
developer experience. Over the coming months we will be working to improve
our performance and observability, while addressing any user-submitted bugs
and feature requests.


### What did the team achieve?

* Finished Rasberry PI demo [#2353](https://github.com/cardano-scaling/hydra/issues/2353)
* Progress on partial fanout [#1468](https://github.com/cardano-scaling/hydra/issues/1468)
* Improved our [HTLC demo](https://github.com/cardano-scaling/hydra-lightning-router/)
* Finished chain-following improvements [#2206](https://github.com/cardano-scaling/hydra/issues/2206)
* Accepted a PR to fix a HTTP endpoint [#2445](https://github.com/cardano-scaling/hydra/pull/2445), thanks [@leuhk!](https://github.com/leuhk)

### What's next?

* Continue looking into memory improvements [#2441](https://github.com/cardano-scaling/hydra/issues/2441), [#2422](https://github.com/cardano-scaling/hydra/issues/2422)
* Improve benchmarking [#2439](https://github.com/cardano-scaling/hydra/issues/2439)
* Investigate bug with incremental commits [#2446](https://github.com/cardano-scaling/hydra/issues/2446)
* Continue work on partial fanout [#1468](https://github.com/cardano-scaling/hydra/issues/1468)
* Release version 1.3.0
